# Story 3.3: Translate the three signature beats into Battle behaviors

Status: done

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

## Story

As a viewer,
I want the tagged beats to drive their specified battle behaviors,
so that the Shaman, Dispel, and Summon are mechanically real, not just narrated. (FR-4)

## Acceptance Criteria

(Verbatim from `_bmad-output/planning-artifacts/epics.md#Story-3.3`, under "### Story 3.3: Translate the three signature beats into Battle behaviors".)

1. **Given** a tagged **Fallen Shaman** **When** the encounter plays **Then** defeated symptom-imps visibly resurrect while the Shaman lives, and when the Shaman is defeated all its imps die in one wave.
2. **Given** a tagged **Dispel** (an assumption Event followed by its tagged ground-truth `Read`) **When** it fires **Then** the on-screen Mirage shatters, the Hero takes a self-inflicted Resolve stagger (wasted effort), the real situation is revealed, and a Scribe-correction signal is emitted (consumed by FR-9).
3. **Given** a tagged **breakthrough** with a charged Insight Gauge **When** it fires **Then** the Eidolon Summon (THUNDORR) is triggered and deals a decisive blow.

### What the gate CAN prove vs what is OPERATOR-verified (READ THIS — split is load-bearing, like 2.3-2.5/3.1)

This story is split exactly like Story 2.4: a **PURE decision layer** (fully gate-provable) + a **THIN Phaser consumer** (the visual reading is operator-verified). Do NOT claim the pixels are proven by the headless gate.

**Gate-PROVABLE (typecheck / lint / test / build) — the bulk of the surface:**
- **WHICH behavior intents + signals fire** for a given `(prev, next, beatsAdvanced, AnnotatedView)` transition: the Shaman live-loop vs one-wave-clear decision; the Dispel's mirage-shatter intent + self-inflicted-Resolve-stagger PRESENTATION cue + reveal + the emitted `scribe-correction` signal; the Summon intent gated on `next.insightGauge` charged + a `summon`-tagged breakthrough. All expressed as plain data, ZERO Phaser, node env (mirrors `animation-plan.test.ts`).
- **R1 holds**: the behavior plan reads the read-only overlay + `BattleState`/`Beat[]` deltas and writes ONLY intents/signals — it imports NO Layer-0 *mechanics value* it could mutate, never returns a `BattleState`, and `pnpm lint` stays green (R1 zones unchanged). A grep proves the plan never writes a `BattleState`/HP/gauge field.
- **The signal is a typed cross-layer output** Story 4.1 (FR-9) can consume: the `scribe-correction` signal carries the Dispel's `groundingPointer` + the firing `cursor` so the Scribe can locate + cross out the prior caption. Proven by the signal's shape + a test that a Dispel transition yields exactly one such signal.
- **Determinism**: same transition + overlay -> byte-identical intents/signals (call twice, `toEqual`).
- **Golden snapshots byte-UNCHANGED** (`src/pace/__snapshots__/` + `src/ingest/__snapshots__/`) — this story adds NO Layer-0 code and reads the fixture without regenerating it; **`grep -ril anthropic dist/` = NOTHING** (R4; behavior layer is SDK-free).

**OPERATOR-verified (NOT a gate — watch `pnpm dev`):** that imps *visibly* resurrect then die as one *readable* wave; that the mirage *visibly* shatters and the Hero *visibly* recoils; that THUNDORR's blow *reads* as decisive; 60fps/no-jank (NFR-1). The polished CINEMATICS (full-scene THUNDORR cutaway; glass-break + record-scratch shatter; simultaneous imp-death wave) are **Stories 3.4/3.5** — this story builds the BEHAVIOR LOGIC + signals + a THIN render wiring on the placeholder cast, NOT the set-pieces.

**Honest reachability gaps (the fixture is a thin redacted dev slice — same posture as 2.4's "rise unreachable at speed=1"):**
- The **Summon positive branch is UNIT-provable only**: the committed `FixtureInterpreter` omits `summon` by design (Story 3.1 — no sub-agent-spawn event in the slice), so no `summon`-tagged beat exists in the real fixture. Prove the Summon branch with a HAND-BUILT `summon` annotation + the fixture's real breakthrough/discharge transition (exactly how 2.4 unit-proved the Hammer-Flurry positive branch with a hand-built multi-source melee). Document it as a gap.
- The **Dispel's "self-inflicted Resolve stagger" is a PRESENTATION cue, not a new mechanic**: in the thin fixture the Dispel's anchor (`u-0002#1`) lands in a *scout* beat (Beat[0]) that drains NO Resolve — the actual wasted-effort Resolve change already lives in the Layer-0 stream (the `counter` at Beat[6] in this fixture; in a richer real session the assumption-then-correction maps to its own Layer-0 drain). The behavior plan emits the recoil CUE; it does NOT compute or mutate Resolve (that would break R1).
- **Imps have NO per-minion HP** in v0.1 `BattleState` (the SAME gap `render-model.ts`/`animation-plan.ts` documented). The Shaman resurrect-loop + one-wave-clear are a PRESENTATION loop keyed on the Shaman's *live-vs-defeated* status (derived from its grounded root-cause Layer-0 event), NOT per-imp HP. No imp HP is invented in `BattleState`.
