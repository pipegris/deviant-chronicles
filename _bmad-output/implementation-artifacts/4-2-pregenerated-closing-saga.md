# Story 4.2: Pre-generated closing Saga

Status: done

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

## Story

As a viewer,
I want a lush closing Saga displayed at victory, authored once offline and baked into the bundle,
so that the demo ends on elegiac Tolkien prose with zero runtime LLM cost. (FR-10, NFR-5)

## Acceptance Criteria

(Verbatim from `_bmad-output/planning-artifacts/epics.md`, under "### Story 4.2: Pre-generated closing Saga".)

**AC1 ‚Äî Offline authoring ‚Üí baked into the bundle**
**Given** `scripts/scribe-saga.ts`
**When** it runs offline
**Then** it authors one Saga via `claude-opus-4-8` (Tolkien-register prompt) over the relevant Event window and bakes it into the `ReplayBundle`.

**AC2 ‚Äî Offline-at-replay display (no runtime LLM)**
**Given** the Replay at the victory milestone
**When** it reaches the closing
**Then** it displays the stored Saga with no runtime LLM call (offline-at-replay).

### Acceptance split (gate-verifiable vs operator-verified) ‚Äî READ THIS FIRST

This story builds the closing-Saga MACHINERY and tests **every** verifiable behavior against a **MOCKED** Anthropic client ‚Äî **zero real network calls, no real `ANTHROPIC_API_KEY`** in dev or CI. The ONE real `claude-opus-4-8` Saga bake over the real Story 10.1 session is a **deferred operator step** (Epic 5, Story 5.2 bundle assembly) and is explicitly NOT part of this story's gate. This is the SAME established decision as Story 3.2 (the frozen interpreter): the authoring script is built + tested against a fake injected client; the single real call is the operator's.

- **Gate-verifiable (Vitest / typecheck / lint / build ‚Äî MUST be green):**
  - **AC1 (the authoring wiring, MOCKED):** with a **fake Anthropic client injected** into `SagaAuthor`, `authorSaga(events)` builds a `claude-opus-4-8` request (right `model`, a Tolkien-register system prompt, the event window as the user message) and returns the canned Saga text; the thin `scripts/scribe-saga.ts` glue maps that text into the `ReplayBundle.saga` seam. R4 isolation is proven by `grep -ril anthropic dist/` finding NOTHING + a source-grep that `scribe/saga.ts` (the browser reader) is SDK-free.
  - **AC2 (the reader + offline-at-replay):** the SDK-free `scribe/saga.ts` reader returns `ReplayBundle.saga` (and handles the not-yet-authored `null` case), and the boot displays it at the victory milestone (`BattleState.victory` latch) via a one-way `RenderPort.renderSaga?` command, with **NO LLM call** on the browser path (the reader only reads the baked string).
  - **Determinism:** a baked-then-loaded Saga is **byte-stable** (the Saga is a plain string in the bundle JSON; the reader returns it verbatim).
- **Operator-verified (manual `pnpm dev`, NOT gated):**
  - The on-screen victory Saga PANEL legibility / typography (readable lush prose, the elegiac scroll), exactly as Stories 2.4/3.4/3.5/4.1 split their on-screen visual acceptance. jsdom does not advance Phaser tweens, so the visual is operator-only.

There is nothing in the gate-verifiable set a human must eyeball. If a test "can't" prove something, the seam is wrong ‚Äî make the client injectable / the reader pure, not the test.
